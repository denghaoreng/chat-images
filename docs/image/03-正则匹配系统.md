# 正则匹配系统

## 概述

正则匹配系统是聊天图片插件的核心功能。当 AI 回复消息后，插件自动扫描消息文本，使用用户预设的正则表达式进行匹配，如果匹配成功，则根据绑定的图片和权重随机选择一张图片展示。

---

## 数据结构

每条规则包含以下字段：

```javascript
{
    "id": "rule_001",                    // 规则唯一 ID
    "name": "战斗场景",                   // 规则名称（用户可识别）
    "regex": "攻击|战斗|斩|劈|魔法|技能", // 正则表达式
    "enabled": true,                     // 是否启用
    "images": [                          // 绑定的图片列表
        {
            "id": "img_001",             // 图片唯一 ID
            "filename": "sword_attack.png", // 文件名
            "weight": 50,                // 权重（0-100）
            "name": "剑术攻击"
        },
        {
            "id": "img_002",
            "filename": "magic_fire.png",
            "weight": 30,
            "name": "火焰魔法"
        },
        {
            "id": "img_003",
            "filename": "critical_hit.png",
            "weight": 20,
            "name": "暴击"
        }
    ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 规则唯一标识，用 `uuidv4()` 生成 |
| `name` | `string` | 规则名称，方便用户识别 |
| `regex` | `string` | 正则表达式字符串（不含分隔符） |
| `enabled` | `boolean` | 是否启用该规则 |
| `images[]` | `array` | 绑定的图片列表 |
| `images[].id` | `string` | 图片唯一标识 |
| `images[].filename` | `string` | 图片文件名（存储在 `images/` 目录下） |
| `images[].weight` | `number` | 权重值（0~100），越高越容易被选中 |
| `images[].name` | `string` | 图片名称/描述 |

## 核心匹配逻辑

### 1. 消息获取

当收到 AI 回复时，从事件数据中提取消息文本：

```javascript
function onMessageReceived(data) {
    // data 为消息对象，包含 id, name, mes, is_user 等字段
    const messageText = data.mes;
    if (!messageText) return;

    // 只处理 AI 消息，不处理用户消息
    if (data.is_user) return;

    // 执行匹配
    matchAndShowImage(messageText);
}
```

### 2. 正则匹配

遍历所有启用的规则，对消息文本进行正则测试：

```javascript
function matchAndShowImage(text) {
    const rules = getRules();

    for (const rule of rules) {
        if (!rule.enabled) continue;

        try {
            const regex = new RegExp(rule.regex, 'gi');
            if (regex.test(text)) {
                // 匹配成功，从该规则的图片列表中随机选择一张
                const selectedImage = selectImageByWeight(rule.images);
                if (selectedImage) {
                    showImage(selectedImage);
                    break; // 只展示第一个匹配规则的图片
                }
            }
        } catch (e) {
            console.error('聊天图片插件: 正则执行错误', rule.regex, e);
        }
    }
}
```

### 3. 权重随机算法

根据图片的权重值进行加权随机选择，权重越高的图片被选中的概率越大：

```javascript
/**
 * 根据权重从图片列表中随机选择一张
 * @param {Array} images - 图片列表 [{filename, weight, ...}]
 * @returns {Object|null} 选中的图片对象
 */
function selectImageByWeight(images) {
    if (!images || images.length === 0) return null;

    // 计算总权重
    const totalWeight = images.reduce((sum, img) => sum + (img.weight || 1), 0);
    if (totalWeight <= 0) {
        // 所有权重为 0 时，等概率随机
        return images[Math.floor(Math.random() * images.length)];
    }

    // 生成随机数
    let random = Math.random() * totalWeight;

    // 遍历图片，累加权重，找到选中项
    for (const img of images) {
        random -= (img.weight || 1);
        if (random <= 0) {
            return img;
        }
    }

    // 兜底返回最后一张
    return images[images.length - 1];
}
```

#### 算法示例

假设有三张图片，权重分别为 50、30、20：

| 图片 | 权重 | 概率 |
|------|------|------|
| 剑术攻击.png | 50 | 50% |
| 火焰魔法.png | 30 | 30% |
| 暴击.png | 20 | 20% |

当 `Math.random() * 100` 结果为：
- `0 ~ 50` → 选中"剑术攻击"
- `50 ~ 80` → 选中"火焰魔法"
- `80 ~ 100` → 选中"暴击"

## 正则表达式编写指南

### 基础语法

- **多关键词匹配**: `关键词1|关键词2|关键词3`
- **忽略大小写**: 匹配时使用 `gi` 标志，自动忽略大小写
- **字符转义**: 特殊字符需转义，如 `\.` `\+` `\*` `\(` `\)`

### 示例

```
# 表情相关
微笑|大笑|苦笑|微笑|咧嘴

# 动作相关
点头|摇头|挥手|鞠躬|跪下|站起来

# 情感相关
开心|悲伤|愤怒|恐惧|惊讶|厌恶|傲慢

# 天气场景
下雨|打雷|闪电|晴天|阴天|刮风

# 战斗场景
攻击|防御|闪避|格挡|反击|冲锋|撤退
```

## 匹配策略

### 单次匹配

当前设计：**只展示第一个匹配规则**的图片（按规则列表的顺序）。

```
用户消息: "他拔出剑，施展了火焰魔法！"
                        │
                        ▼
规则1: "攻击|战斗|斩|劈|魔法|技能" → ✅ 匹配
                        │
                        ▼
                随机选择图片并展示
                        │
                        ▼
                停止继续匹配（break）
```

### 可选扩展

未来可以增加以下匹配策略选项（可在设置中添加开关）：

| 策略 | 说明 |
|------|------|
| **首规则优先**（当前） | 按顺序匹配第一个成功的规则，展示其绑定的图片 |
| **全部匹配** | 所有匹配成功的规则各自选择一张图片，全部展示 |
| **随机匹配** | 从所有匹配成功的规则中随机选一个展示 |

---

## 注意事项

1. **正则语法验证**: 用户输入正则时，应在保存前进行语法验证，防止无效正则会阻塞匹配
2. **性能考虑**: 避免过于复杂的正则表达式，可能影响匹配性能
3. **空文本处理**: 消息为空或不包含文本时，跳过匹配
4. **多消息匹配**: 考虑流式生成场景，等待完整消息生成后再进行匹配
