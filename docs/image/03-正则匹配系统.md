# 正则匹配系统

## 概述

正则匹配系统是聊天图片插件的核心功能，代码位于 `matcher.js` 中。当 AI 回复消息后，插件自动扫描消息文本，使用用户预设的正则表达式进行匹配，支持多规则集顺序匹配、加权随机选图、批次队列展示。

**核心流程**：匹配入口 → 获取启用规则 → 按规则集分组 → 逐规则匹配 → 加权随机选图 → 批次队列展示

---

## 数据结构

当前规则数据结构增加了规则集、展开状态等字段：

```javascript
{
    "id": "rule_abc123",              // 规则唯一 ID
    "name": "战斗场景",                // 规则名称
    "regex": "攻击|战斗|斩|劈|魔法",   // 正则表达式
    "enabled": true,                   // 是否启用
    "order": 0,                        // 排序优先级
    "ruleSetId": "",                   // 所属规则集 ID
    "duration": 5,                     // 图片停留秒数（0=永久）
    "_expanded": true,                 // 界面折叠状态
    "images": [                        // 绑定的图片列表
        {
            "id": "img_001",
            "filename": "sword_attack.png",
            "weight": 50,
            "name": "剑术攻击"
        }
    ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 规则唯一标识，用 `generateId('rule')` 生成 |
| `name` | `string` | 规则名称 |
| `regex` | `string` | 正则表达式字符串（不含分隔符） |
| `enabled` | `boolean` | 是否启用 |
| `order` | `number` | 匹配优先级，越小越靠前 |
| `duration` | `number` | 每张图片停留秒数（0=永久） |
| `ruleSetId` | `string` | 所属规则集 ID |
| `_expanded` | `boolean` | 界面折叠状态 |
| `images[]` | `array` | 绑定的图片列表 |
| `images[].weight` | `number` | 权重值（0~100） |

## 核心匹配逻辑 (matcher.js)

### 1. 入口：performMatch()

匹配入口，获取所有启用的规则，按规则集分组，按规则集顺序逐规则匹配：

```javascript
export function performMatch(messageText) {
    const data = getRulesData();
    const enabledRules = data.rules.filter(r => r.enabled);
    if (enabledRules.length === 0) return [];

    // 按规则集分组
    const ruleSetOrder = data.ruleSets.map(rs => rs.id);
    const grouped = {};
    enabledRules.forEach(rule => {
        const setId = rule.ruleSetId || '';
        if (!grouped[setId]) grouped[setId] = [];
        grouped[setId].push(rule);
    });

    // 按规则集顺序 + 规则 order 排序
    const matchedResults = [];
    const seenIds = new Set();

    // 先处理有规则集的规则
    ruleSetOrder.forEach(setId => {
        const rules = (grouped[setId] || []).sort((a, b) => a.order - b.order);
        for (const rule of rules) {
            if (seenIds.has(rule.id)) continue;
            const match = matchSingleRule(rule, messageText);
            if (match) {
                matchedResults.push(match);
                seenIds.add(rule.id);
            }
        }
    });

    // 再处理无规则集的规则
    const noSetRules = (grouped[''] || []).sort((a, b) => a.order - b.order);
    for (const rule of noSetRules) {
        if (seenIds.has(rule.id)) continue;
        const match = matchSingleRule(rule, messageText);
        if (match) {
            matchedResults.push(match);
            seenIds.add(rule.id);
        }
    }

    return matchedResults;
}
```

### 2. 单规则匹配：matchSingleRule()

```javascript
export function matchSingleRule(rule, text) {
    try {
        const regex = sanitizeRegex(rule.regex);
        if (!regex) return null;
        const re = new RegExp(regex, 'gi');
        if (re.test(text)) {
            const selected = selectImageByWeight(rule.images);
            if (selected) {
                return { rule, image: selected };
            }
        }
    } catch (e) {
        console.warn('聊天图片插件: 正则匹配失败', rule.regex, e);
    }
    return null;
}
```

### 3. 正则清理：sanitizeRegex()

移除非打印字符和首尾空白：

```javascript
export function sanitizeRegex(str) {
    if (!str) return '';
    return str.replace(/[\x00-\x1f]/g, '').trim();
}
```

### 4. 加权随机选图：selectImageByWeight()

```javascript
export function selectImageByWeight(images) {
    if (!images || images.length === 0) return null;

    const totalWeight = images.reduce((sum, img) => sum + (img.weight || 1), 0);
    if (totalWeight <= 0) {
        return images[Math.floor(Math.random() * images.length)];
    }

    let random = Math.random() * totalWeight;
    for (const img of images) {
        random -= (img.weight || 1);
        if (random <= 0) return img;
    }
    return images[images.length - 1];
}
```

#### 算法示例

| 图片 | 权重 | 概率 |
|------|------|------|
| 剑术攻击.png | 50 | 50% |
| 火焰魔法.png | 30 | 30% |
| 暴击.png | 20 | 20% |

随机值 `0~50` → 剑术攻击，`50~80` → 火焰魔法，`80~100` → 暴击

### 5. 图片队列展示：queueBatchesForMessage()

支持按规则集分组、每张图片停留指定秒数的队列展示：

```javascript
export function queueBatchesForMessage(matchedResults) {
    if (!matchedResults || matchedResults.length === 0) return;

    // 按规则集分组
    const grouped = {};
    matchedResults.forEach(item => {
        const setId = item.rule.ruleSetId || '__noset__';
        if (!grouped[setId]) grouped[setId] = [];
        grouped[setId].push(item);
    });

    // 按规则集顺序展示
    // 每组内：先显示图片，duration 秒后切换到下一张
    let totalDelay = 0;
    Object.values(grouped).forEach(batch => {
        batch.forEach((item, index) => {
            const duration = (item.rule.duration || 5) * 1000;
            setTimeout(() => {
                renderImagesInDom([item.image], item.rule.name);
            }, totalDelay);
            // 最后一张的停留由下一个批次接管
            if (index < batch.length - 1) {
                totalDelay += duration;
            }
        });
        // 批次之间额外等待最后一张的 duration
        if (batch.length > 0) {
            totalDelay += (batch[batch.length - 1].rule.duration || 5) * 1000;
        }
    });
}
```

### 消息处理入口 (index.js)

```javascript
function onMessageReceived(data) {
    if (data.is_user) return;
    const text = data.mes;
    if (!text) return;
    const results = performMatch(text);
    if (results.length > 0) {
        queueBatchesForMessage(results);
    }
}
```

## 正则表达式编写指南

### 基础语法

- **多关键词匹配**: `关键词1|关键词2|关键词3`
- **忽略大小写**: 匹配时使用 `gi` 标志，自动忽略大小写
- **字符转义**: 特殊字符需转义，如 `\.` `\+` `\*` `\(` `\)`

### 示例

```
# 表情相关
微笑|大笑|苦笑|微笑|咧嘴

# 动作相关
点头|摇头|挥手|鞠躬|跪下|站起来

# 情感相关
开心|悲伤|愤怒|恐惧|惊讶|厌恶|傲慢

# 天气场景
下雨|打雷|闪电|晴天|阴天|刮风

# 战斗场景
攻击|防御|闪避|格挡|反击|冲锋|撤退
```

## 匹配策略

### 单次匹配

当前设计：**只展示第一个匹配规则**的图片（按规则列表的顺序）。

```
用户消息: "他拔出剑，施展了火焰魔法！"
                        │
                        ▼
规则1: "攻击|战斗|斩|劈|魔法|技能" → ✅ 匹配
                        │
                        ▼
                随机选择图片并展示
                        │
                        ▼
                停止继续匹配（break）
```

### 可选扩展

未来可以增加以下匹配策略选项（可在设置中添加开关）：

| 策略 | 说明 |
|------|------|
| **首规则优先**（当前） | 按顺序匹配第一个成功的规则，展示其绑定的图片 |
| **全部匹配** | 所有匹配成功的规则各自选择一张图片，全部展示 |
| **随机匹配** | 从所有匹配成功的规则中随机选一个展示 |

## 注意事项

1. **正则语法验证**: 用户输入正则时，应在保存前进行语法验证，防止无效正则会阻塞匹配
2. **性能考虑**: 避免过于复杂的正则表达式，可能影响匹配性能
3. **空文本处理**: 消息为空或不包含文本时，跳过匹配
4. **多消息匹配**: 考虑流式生成场景，等待完整消息生成后再进行匹配
