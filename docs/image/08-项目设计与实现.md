# 聊天图片插件 — 项目设计与实现

> 编写日期：2026年5月28日

## 一、项目概述

聊天图片插件是 SillyTavern 的一个前端扩展，核心功能是根据 AI 回复内容中的正则表达式匹配，自动插入预设图片到聊天消息中。采用纯前端实现，依赖 SillyTavern 内置 API，无需服务端插件。

### 功能总览

| 模块 | 功能 | 状态 |
|------|------|------|
| 导航栏入口 | 顶部导航栏 Drawer 抽屉按钮 | ✅ |
| 角色图片集 | 按角色分组管理规则集 | ✅ |
| 规则集 | 规则的分组管理，绑定到角色集 | ✅ |
| 规则管理 | 正则表达式 + 图片绑定 + 权重 + 停留时间 | ✅ |
| 单张上传 | 选择图片上传到服务端 | ✅ |
| 批量上传 | 多张图片批量上传到一条规则 | ✅ |
| 批量创建规则 | 多张图片批量创建多条规则（弹窗） | ✅ |
| 正则匹配 | AI 回复后自动匹配规则 | ✅ |
| 图片队列 | 按规则集/规则顺序排队展示图片 | ✅ |
| 图片放大 | 双击/点击放大镜按钮全屏查看图片 | ✅ |
| 折叠记忆 | 规则折叠状态持久化到设置 | ✅ |
| 标签记忆 | 刷新页面前后保留最后使用的标签页 | ✅ |
| 展开/折叠全部 | 一键展开或折叠所有规则 | ✅ |
| 正则手册 | 内置正则表达式语法参考文档 | ✅ |

## 三、核心架构设计

### 3.1 数据模型

插件数据存储在 `extensionSettings[MODULE_NAME]` 中，通过 `getContext().saveSettingsDebounced()` 持久化。

```
currentSettings (extensionSettings['chat-images'])
├── enabled: boolean          # 总开关
├── autoDetect: boolean       # 自动检测开关
├── showInChat: boolean       # 聊天中展示开关
└── rulesData:
    ├── version: number
    ├── charSets: []           # 角色图片集
    │   └── { id, name, enabled }
    ├── ruleSets: []           # 规则集
    │   └── { id, name, order, enabled, charSetId }
    └── rules: []              # 规则
        └── { id, name, regex, enabled, order, duration, ruleSetId, images[], _expanded }
```

**关键设计决策：所有规则数据统一存储在 `extensionSettings` 中，不需要独立的 JSON 文件。** 这样利用 SillyTavern 的内置同步机制，多端自动同步。

### 3.2 导航栏系统

实现了 SillyTavern 标准的 Drawer 抽屉导航栏：

```
#top-settings-holder（flex 容器）
  ├── #ai-config-button        AI响应配置
  ├── #sys-settings-button     系统设置
  ├── #advanced-formatting-button  高级格式化
  ├── #WI-SP-button            世界书
  ├── #chat-images-drawer      聊天图片 ← 插件插入在这里
  ├── #user-settings-button    用户设定管理
  ├── #backgrounds-button      背景管理
  ├── #extensions-settings-button  扩展管理
  ├── #persona-management-button   角色管理
  └── #rightNavHolder          角色选择
```

**插入方式（两步法）：**
```javascript
$('#top-settings-holder').append(drawerHtml);          // 先添加到末尾
$('#chat-images-drawer').insertBefore('#user-settings-button'); // 再移到目标位置
```

**交互机制：** 点击 `.drawer-toggle` → 调用 `doNavbarIconClick()` → 切换 `closedDrawer/openDrawer` 和 `closedIcon/openIcon` 类。

### 3.3 正则匹配系统

**匹配流程：**

```
AI回复 → onMessageReceived → performMatch(text)
  → 获取启用规则（getEnabledRules）
  → 按规则集分组排序
  → 对每条规则执行 matchSingleRule
    → sanitizeRegex(rule.regex)      # 净化用户输入的正则
    → new RegExp(pattern, 'gi')      # 创建正则（全局+忽略大小写）
    → regex.test(text)                # 测试是否匹配
  → 匹配成功则加权随机选择图片（selectImageByWeight）
  → 按规则集批次入队（queueBatchesForMessage）
  → 逐张展示图片，支持停留时间
```

**sanitizeRegex 设计：** 支持解析 `/pattern/gi` 定界符格式（兼容从其他工具复制的正则），也支持直接写模式（无定界符）。

**加权随机算法：** 所有权重求和，产生随机数，按权重比例选择图片。

### 3.4 图片队列系统

图片展示使用队列机制，支持：

1. **规则集批次** — 按规则集顺序逐批处理
2. **规则顺序** — 同一规则集内按 order 排序
3. **停留时间** — 每张图片可设置停留秒数（0=永久）
4. **定时器管理** — 每个消息独立的定时器，滑动/重新渲染时自动清理

### 3.5 图片放大功能

自定义实现的全屏图片查看器，不依赖 SillyTavern 的 `extra.media` 系统：

```javascript
// 创建全屏遮罩覆盖层
const overlay = document.createElement('div');
overlay.style.cssText = 'position:fixed;top:0;left:0;width:100vw;height:100dvh;...';

// 创建大图
const enlargedImg = document.createElement('img');
enlargedImg.src = imgEl.src;
enlargedImg.style.cssText = 'max-width:95vw;max-height:95dvh;...';
```

**关键设计：** 使用 `100dvh`（动态视口高度）解决移动端浏览器地址栏伸缩问题。图片存储在 `extra.chatImages` 而非 `extra.media`，避免被发送给 AI 模型。

## 四、技术难点与解决方案

### 难点 1：弹窗 DOM 销毁导致数据丢失

**问题：** `callGenericPopup` 在弹窗关闭后会销毁弹窗 DOM，导致在 `await popup` 之后读取表单值时返回 `undefined`。

**解决方案：** 使用 `input` 事件实时追踪表单值到内存变量中：

```javascript
let formPrefix = '批量规则';
let formDuration = 2;
$(document).on('input', '#batch-rule-prefix', function() { formPrefix = ... });
$(document).on('input', '#batch-rule-duration', function() { formDuration = ... });
// await popup 之后使用 formXxx 变量，而非从 DOM 读取
```

### 难点 2：导航栏入口位置控制

**问题：** 最初使用 `$(drawerHtml).insertBefore('#user-settings-button')` 直接插入，但用户反馈入口消失。

**根因：** 目标元素 `#user-settings-button` 可能在插件初始化时尚未加载完毕？实际上是因为用 `insertBefore` + HTML 字符串在某些 jQuery 版本中可能不按预期工作。

**解决方案（两步法）：**
1. 先用 `append` 确保入口一定出现
2. 再用 `insertBefore` 移动到目标位置

### 难点 3：图片上传后事件失效

**问题：** 上传图片后，新渲染的图片删除按钮无法点击。

**根因：** `imagesContainer.html(renderRuleImages(rule))` 替换了 DOM 元素，但事件绑定在旧元素上。

**解决方案：** 将直接事件绑定改为**事件委托**：
```javascript
// 改为：稳定父元素上委托
ruleElement.on('click', '.rule-image-delete', async function() { ... });
ruleElement.on('input', '.image-weight-slider', function() { ... });
```

### 难点 4：图片放大不发送给模型

**问题：** 使用 SillyTavern 原生的 `mes_media_enlarge` 机制需要将图片加入 `extra.media`，但这会导致图片被发送给 AI 模型（在 `openai.js` 中有 `const media = chat[j]?.extra?.media`）。

**解决方案：** 
1. 图片存储在 `extra.chatImages` 而非 `extra.media`
2. 自定义全屏放大覆盖层，完全独立于 SillyTavern 的 `expandMessageMedia`
3. 移动端兼容：使用 `100dvh` 单位 + 双击检测（`click` 事件时间差模拟 `dblclick`）

### 难点 5：移动端双击检测

**问题：** 移动端浏览器不支持 `dblclick` 事件。

**解决方案：** 用 `click` 事件配合时间戳检测：
```javascript
const now = Date.now();
const lastTap = img._lastTap || 0;
if (now - lastTap < 400) {
    chatImageEnlarge(img);  // 400ms 内两次点击视为双击
    img._lastTap = 0;
} else {
    img._lastTap = now;
}
```

### 难点 6：规则折叠状态持久化

**问题：** 关闭弹窗再打开后，已展开的规则被重置为折叠状态。

**解决方案：** 在规则数据中添加 `_expanded` 字段：
- 渲染时根据 `rule._expanded` 决定初始状态
- 点击折叠按钮时通过 `updateRule` 持久化
- 新规则默认 `_expanded: true`

### 难点 7：标签页记忆（lastActiveTab）

**问题：** 刷新页面后首次打开默认显示角色集，但用户切换到其他标签后关闭再打开，应该保持上次的标签。

**解决方案：** 使用内存变量 `lastActiveTab`：
- 每次标签点击时记录 `lastActiveTab = tab`
- 打开弹窗时：`lastActiveTab` 有值则恢复，无值则默认角色集
- 编辑按钮跳转时也要更新 `lastActiveTab`

## 六、性能考虑

1. **规则列表渲染** — 每次渲染全部重建 DOM，但规则数量通常在几十条级别，性能可接受
2. **正则匹配** — 每条规则独立创建 `RegExp` 对象，避免 `lastIndex` 问题
3. **图片队列定时器** — 每个消息独立管理定时器，滑动/切换时自动清理
4. **事件委托** — 图片列表使用事件委托避免重复绑定
