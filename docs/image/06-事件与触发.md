# 事件与触发

## 概述

聊天图片插件的核心触发逻辑是：**AI 回复消息后**，自动检测消息文本并执行正则匹配。这需要通过 SillyTavern 的事件系统来监听相应的消息事件。

---

## 事件选择

SillyTavern 提供了多个与消息相关的事件，聊天图片插件主要关注以下事件：

| 事件 | 触发时机 | 推荐用途 |
|------|----------|----------|
| `MESSAGE_RECEIVED` | LLM 消息已生成，已记录但**未渲染**到 UI | ✅ 主要匹配时机 |
| `CHARACTER_MESSAGE_RENDERED` | LLM 消息已渲染到 UI | 备选（需要等待渲染完成） |
| `MESSAGE_SWIPED` | 用户滑动切换回复 | 切换回复时重新匹配 |
| `STREAM_TOKEN_RECEIVED` | 流式生成中收到新 token | ❌ 不适合（需要完整消息） |

### 推荐：监听 MESSAGE_RECEIVED

`MESSAGE_RECEIVED` 是最合适的监听事件，因为此时消息已经生成完毕且包含完整文本。

```javascript
import { getContext } from '../../../extensions.js';
const { eventSource, event_types } = getContext();

// 在 init 中注册事件
export function init() {
    loadSettings();
    loadRulesData();
    addNavBarDrawer();

    // 监听 AI 消息接收事件
    eventSource.on(event_types.MESSAGE_RECEIVED, onMessageReceived);

    // 监听滑动切换回复
    eventSource.on(event_types.MESSAGE_SWIPED, onMessageSwiped);
}
```

## 消息处理逻辑

### 1. 消息接收处理器

```javascript
/**
 * AI 消息接收时的处理函数
 * @param {Object} data - 消息数据
 */
function onMessageReceived(data) {
    // 检查插件是否启用
    if (!currentSettings.enabled || !currentSettings.autoDetect) return;

    // 只处理 AI 消息（不是用户消息也不是系统消息）
    if (data.is_user || data.is_system) return;

    const messageText = data.mes;
    if (!messageText) return;

    // 执行正则匹配
    performMatch(messageText);
}
```

### 2. 滑动切换处理器

```javascript
/**
 * 用户滑动切换回复时的处理函数
 * @param {Object} data - 包含当前消息 ID 的数据
 */
function onMessageSwiped(data) {
    if (!currentSettings.enabled) return;

    // 获取当前显示的消息内容
    const { chat } = SillyTavern.getContext();
    const currentMessage = chat[chat.length - 1];
    if (!currentMessage || currentMessage.is_user) return;

    performMatch(currentMessage.mes);
}
```

### 3. 匹配执行函数

```javascript
/**
 * 执行正则匹配并展示图片
 * @param {string} text - 要匹配的消息文本
 */
function performMatch(text) {
    if (!text) return;

    const enabledRules = getEnabledRules();
    if (enabledRules.length === 0) return;

    const matchedImages = [];

    for (const rule of enabledRules) {
        try {
            const regex = new RegExp(rule.regex, 'gi');
            if (regex.test(text)) {
                // 从该规则中按权重选择图片
                const selectedImage = selectImageByWeight(rule.images);
                if (selectedImage) {
                    matchedImages.push({
                        rule: rule,
                        image: selectedImage,
                    });

                    // "首个规则优先"策略：找到第一个匹配就停止
                    if (currentSettings.matchStrategy === 'first') {
                        break;
                    }
                }
            }
        } catch (e) {
            console.error(`聊天图片插件: 规则 "${rule.name}" 的正则执行错误`, e);
        }
    }

    // 处理匹配结果
    if (matchedImages.length > 0) {
        handleMatchedImages(matchedImages);
    }
}
```

### 4. 结果处理

```javascript
/**
 * 处理匹配到的图片
 * @param {Array} matchedImages - 匹配到的图片列表
 */
function handleMatchedImages(matchedImages) {
    if (matchedImages.length === 0) return;

    // 根据匹配策略决定展示哪些图片
    let imagesToShow = [];

    switch (currentSettings.matchStrategy) {
        case 'first':
            // 只展示第一个匹配规则的图片
            imagesToShow = [matchedImages[0].image];
            break;
        case 'all':
            // 展示所有匹配规则的图片
            imagesToShow = matchedImages.map(m => m.image);
            break;
        case 'random':
            // 从所有匹配结果中随机选一个
            const randomIndex = Math.floor(Math.random() * matchedImages.length);
            imagesToShow = [matchedImages[randomIndex].image];
            break;
        default:
            imagesToShow = [matchedImages[0].image];
    }

    // 展示图片
    for (const image of imagesToShow) {
        displayImage(image);
    }
}
```

                     │
                     ▼

                     │
                     ▼
┌─────────────────────────────────────────────────┐
│           onMessageReceived(data)                │
│   ┌─────────────────────────────────────────┐   │
│   │ 1. 检查插件是否启用                      │   │
│   │ 2. 检查是否为 AI 消息（非用户/系统）      │   │

## 图片展示函数

### 1. 居中模态框弹窗

```javascript
/**
 * 在居中模态框中展示图片
 * @param {Object} image - 图片元数据
 */
function showImageModal(image) {
    const modal = document.getElementById('chat-images-modal');
    const modalImg = document.getElementById('chat-images-modal-img');

    if (!modal || !modalImg) return;

    // 设置图片
    modalImg.src = getImageUrl(image.filename);
    modalImg.alt = image.name || '聊天图片';

    // 显示模态框
    modal.classList.add('open');

    // 自动关闭
    if (currentSettings.modalAutoClose) {
        setTimeout(() => {
            modal.classList.remove('open');
        }, currentSettings.modalAutoCloseDelay);
    }
}

// 关闭模态框
function closeImageModal() {
    const modal = document.getElementById('chat-images-modal');
    if (modal) modal.classList.remove('open');
}
```

### 2. 插入到聊天

```javascript
/**
 * 将图片作为消息插入到聊天中
 * @param {Object} image - 图片元数据
 */
function insertImageToChat(image) {
    const { chat, addOneMessage, name1 } = SillyTavern.getContext();

    // 构建图片消息
    const imageMessage = {
        name: name1,
        is_user: true,
        is_system: false,
        send_date: Date.now(),
        mes: `[图片] ${image.name || ''}`,
        extra: {
            image: image.filename,
            type: 'chat-image',
        },
    };

    // 添加到聊天
    addOneMessage(imageMessage);

    // 或使用 HTML 图片标签
    const imageHtmlMessage = {
        name: name1,
        is_user: true,
        is_system: false,
        send_date: Date.now(),
        mes: `<img src="${getImageUrl(image.filename)}" alt="${image.name || '聊天图片'}" class="chat-image">`,
    };

    addOneMessage(imageHtmlMessage);
}
```

## 其他相关事件

除了核心的匹配触发事件外，还可以监听以下事件来完善插件功能：

### 设置变更时重新加载

```javascript
// 当设置更新时重新加载
eventSource.on(event_types.SETTINGS_UPDATED, () => {
    loadSettings();
});

// 聊天切换时重置状态
eventSource.on(event_types.CHAT_CHANGED, () => {
    closeImageModal();
});
```

### 生命周期钩子

```javascript
// 插件禁用时清理
export function onDisable() {
    closeImageModal();
    // 移除事件监听
    const { eventSource, event_types } = SillyTavern.getContext();
    eventSource.removeListener(event_types.MESSAGE_RECEIVED, onMessageReceived);
    eventSource.removeListener(event_types.MESSAGE_SWIPED, onMessageSwiped);
}

// 插件启用时重新注册
export function onEnable() {
    const { eventSource, event_types } = SillyTavern.getContext();
    eventSource.on(event_types.MESSAGE_RECEIVED, onMessageReceived);
    eventSource.on(event_types.MESSAGE_SWIPED, onMessageSwiped);
}

// 插件删除时清理数据
export async function onDelete() {
    // 清理 DOM 元素
    $('#chat-images-drawer').remove();
    $('#chat-images-modal').remove();
}
```

## 重要注意事项

1. **流式生成**: 流式生成时 `MESSAGE_RECEIVED` 在完整消息生成完成后才触发，所以无需处理中间 token
2. **重复匹配**: 确保同一消息不会触发多次匹配（可使用消息 ID 去重）
3. **异步处理**: 图片加载是异步的，确保在图片加载完成后再展示
4. **资源清理**: 模态框关闭时清理图片资源，避免内存泄漏
5. **错误容错**: 单个规则的正则错误不应影响其他规则的执行
