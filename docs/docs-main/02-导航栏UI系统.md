# SillyTavern 导航栏 UI 系统

## 一、概述

SillyTavern 的导航栏（Top Settings Bar）是插件添加快捷入口的主要位置。导航栏位于页面顶部，由一系列**抽屉（Drawer）**组成，每个抽屉包含一个图标按钮和一个可展开的面板。

本文档详细说明导航栏的 HTML 结构、CSS 布局、交互机制，以及插件如何正确插入和管理导航栏入口。

---

## 二、HTML 结构

### 2.1 容器

导航栏的所有抽屉都位于 `#top-settings-holder` 容器内：

```html
<div id="top-settings-holder">
    <!-- 所有 drawer 放在这里 -->
</div>
```

### 2.2 抽屉（Drawer）的标准结构

每个抽屉遵循统一的 HTML 结构：

```html
<div id="[drawer-id]" class="drawer">
    <!-- 切换按钮（图标） -->
    <div class="drawer-toggle">
        <div class="drawer-icon fa-solid fa-[图标名] fa-fw closedIcon"
             title="提示文本"></div>
    </div>
    <!-- 展开面板 -->
    <div id="[panel-id]" class="drawer-content closedDrawer">
        <!-- 面板内容 -->
    </div>
</div>
```

**各部分的含义：**

| 元素 | 类/属性 | 作用 |
|------|---------|------|
| 外层 div | `class="drawer"` | 标识这是一个导航抽屉容器 |
| 切换按钮 | `class="drawer-toggle"` | 点击此区域展开/收起面板 |
| 图标 | `class="drawer-icon"` + FontAwesome 类 | 显示的图标 |
| 图标状态 | `closedIcon` / `openIcon` | 收起/展开状态的样式 |
| 面板 | `class="drawer-content"` | 展开后显示的面板内容 |
| 面板状态 | `closedDrawer` / `openDrawer` | 收起/展开状态 |

## 三、CSS 布局

### 3.1 容器布局

```css
#top-settings-holder {
    display: flex;
    margin: 0 auto;
    height: var(--topBarBlockSize);
    justify-content: center;
    z-index: 3005;
    position: relative;
    width: var(--sheldWidth);
}
```

- 使用 **Flexbox** 水平排列
- 居中对齐（`justify-content: center`）
- 所有图标按钮等宽排列

### 3.2 抽屉容器

```css
.drawer {
    align-items: center;
    justify-content: center;
    display: flex;
    flex-flow: row;
    width: 100%;
}
```

### 3.3 图标样式

```css
.drawer-icon {
    display: inline-block;
    cursor: pointer;
    font-size: var(--topBarIconSize);
    padding: 1px 3px;
}

.drawer-icon.closedIcon {
    opacity: 0.3;           /* 收起时半透明 */
    transition: all var(--animation-duration-2x);
}

.drawer-icon.closedIcon:hover {
    opacity: 1;             /* 悬停时高亮 */
}

.drawer-icon.openIcon {
    transition: all var(--animation-duration-2x);
}
```

### 3.4 面板样式

```css
.drawer-content {
    background-color: var(--SmartThemeBlurTintColor);
    border-radius: 10px;
    padding: 5px;
    border: 1px solid var(--SmartThemeBorderColor);
    min-width: 450px;
    width: var(--sheldWidth);
    display: none;              /* 默认隐藏 */
    position: absolute;
    top: var(--topBarBlockSize);
    left: 0;
    right: 0;
    margin: 0 auto;
    height: 0;
    backdrop-filter: blur(...);
}

.drawer-content.openDrawer {
    display: block;
    height: auto;
    overflow-y: auto;
}
```

**关键点：**
- 面板**绝对定位**在导航栏下方（`top: var(--topBarBlockSize)`）
- 默认 `display: none; height: 0`（隐藏）
- 展开时 `display: block; height: auto`（显示）
- `min-width: 450px` 保证面板最小宽度

## 四、交互机制

### 4.1 点击事件处理

导航栏的点击交互由 `doNavbarIconClick()` 函数处理（定义在 `script.js` 中）：

```javascript
export async function doNavbarIconClick() {
    const icon = $(this).find('.drawer-icon');
    const drawer = $(this).parent().find('.drawer-content');
    const drawerWasOpenAlready = $(this).parent().find('.drawer-content').hasClass('openDrawer');
    const targetDrawerID = $(this).parent().find('.drawer-content').attr('id');

    if (!drawerWasOpenAlready) {
        // 1. 关闭所有其他已打开的面板
        const $openDrawers = $('.openDrawer:not(.pinnedOpen)');
        const $openIcons = $('.openIcon:not(.drawerPinnedOpen)');
        for (const iconEl of $openIcons) {
            $(iconEl).toggleClass('closedIcon openIcon');
        }
        for (const el of $openDrawers) {
            $(el).toggleClass('closedDrawer openDrawer');
        }
        // 2. 等待动画完成
        if ($openDrawers.length && animation_duration) {
            await delay(animation_duration);
        }
        // 3. 打开当前面板
        icon.toggleClass('openIcon closedIcon');
        drawer.toggleClass('openDrawer closedDrawer');
    } else {
        // 已打开则切换关闭
        icon.toggleClass('closedIcon openIcon');
        drawer.toggleClass('closedDrawer openDrawer');
    }
}
```

### 4.2 全局事件绑定

`script.js` 在页面初始化时绑定全局事件：

```javascript
$('.drawer-toggle').on('click', doNavbarIconClick);
```

> **注意：** 这个绑定是在页面加载时对已存在的 `.drawer-toggle` 元素生效。插件动态添加的元素不会自动获得此事件绑定，因此插件需要**自行绑定点击事件**。

### 4.3 状态类切换规则

| 元素 | 收起状态 | 展开状态 |
|------|---------|---------|
| 图标 | `closedIcon` | `openIcon` |
| 面板 | `closedDrawer` | `openDrawer` |

## 五、插件如何添加导航栏入口

### 5.1 标准步骤

1. **构造抽屉 HTML**：按照标准 drawer 结构构建
2. **插入到 DOM**：添加到 `#top-settings-holder` 中的合适位置
3. **绑定点击事件**：用 `doNavbarIconClick` 处理展开/收起
4. **绑定关闭按钮**：（可选）面板内的关闭按钮

### 5.2 插入位置的方案对比

#### 方案 A：追加到末尾（最安全）

```javascript
$('#top-settings-holder').append(drawerHtml);
```

- ✅ 始终能工作，无依赖
- ❌ 只能放在最后（最右侧）

#### 方案 B：插入到指定元素之前（推荐）

```javascript
// 两步法：先确保添加，再移动到目标位置
$('#top-settings-holder').append(drawerHtml);
$('#chat-images-drawer').insertBefore('#user-settings-button');
```

- ✅ `append` 保证入口一定出现
- ✅ `insertBefore` 移动到目标位置
- ✅ 即使目标元素不存在，入口也不会丢失

#### 方案 C：使用 insertAdjacentHTML（原生 DOM API）

```javascript
const targetBtn = document.getElementById('user-settings-button');
if (targetBtn) {
    targetBtn.insertAdjacentHTML('beforebegin', drawerHtml);
} else {
    $('#top-settings-holder').append(drawerHtml);
}
```

### 5.3 可用锚点（插入位置参照）

要将插件入口放在特定位置，可使用以下锚点：

| 目标位置 | 锚点选择器 |
|---------|-----------|
| 最左侧（第1位） | `$('#top-settings-holder').prepend(drawerHtml)` |
| AI配置之后 | `$('#ai-config-button').after(drawerHtml)` |
| **用户设定之前** | `$('#user-settings-button').before(drawerHtml)` |
| 扩展管理之前 | `$('#extensions-settings-button').before(drawerHtml)` |
| 角色管理之前 | `$('#persona-management-button').before(drawerHtml)` |
| 最右侧（最后1位） | `$('#top-settings-holder').append(drawerHtml)` |

### 5.4 完整示例代码

以下是一个插件添加导航栏入口的完整示例：

```javascript
function addNavBarDrawer() {
    // 1. 避免重复添加
    if ($('#my-plugin-drawer').length) return;

    // 2. 构造抽屉 HTML
    const drawerHtml = `
    <div id="my-plugin-drawer" class="drawer">
        <div class="drawer-toggle drawer-header">
            <div class="drawer-icon fa-solid fa-star fa-fw closedIcon"
                 title="我的插件"></div>
        </div>
        <div id="my-plugin-panel" class="drawer-content closedDrawer">
            <div class="drawer-content-inner">
                <!-- 面板内容 -->
                <p>这是我的插件面板</p>
            </div>
        </div>
    </div>`;

    // 3. 插入到用户设定管理前面（两步法，安全可靠）
    $('#top-settings-holder').append(drawerHtml);
    $('#my-plugin-drawer').insertBefore('#user-settings-button');

    // 4. 绑定点击事件（必须！否则点击无效）
    $('#my-plugin-drawer .drawer-toggle').on('click', async function() {
        const { doNavbarIconClick } = await import('../../../../script.js');
        doNavbarIconClick.call(this);

        // 打开时执行自定义逻辑
        if ($('#my-plugin-panel').hasClass('openDrawer')) {
            console.log('面板已打开');
            // 刷新数据、渲染列表等
        }
    });

    // 5. （可选）关闭按钮
    $('#my-plugin-close-btn').on('click', function() {
        $('#my-plugin-drawer .drawer-toggle').trigger('click');
    });
}
```

### 5.5 清理（onDelete 钩子）

插件被删除时，应清理自己添加的 DOM 元素：

```javascript
export async function onDelete() {
    $('#my-plugin-drawer').remove();
    // 清理其他相关 DOM
}
```

## 六、插件生命周期与 manifest.json

### 6.1 支持的钩子

| 钩子名 | 触发时机 | 说明 |
|--------|---------|------|
| `activate` | 插件激活时 | 对应 `init()` 函数，在此添加导航栏入口 |
| `install` | 首次安装 | 对应 `onInstall()`，执行初始化设置 |
| `delete` | 插件被删除 | 对应 `onDelete()`，清理 DOM |
| `enable` | 插件启用 | 对应 `onEnable()`，重新绑定事件 |
| `disable` | 插件禁用 | 对应 `onDisable()`，解绑事件 |

### 6.2 manifest.json 示例

```json
{
    "display_name": "我的插件",
    "loading_order": 10,
    "js": "index.js",
    "css": "style.css",
    "hooks": {
        "activate": "init",
        "install": "onInstall",
        "delete": "onDelete",
        "enable": "onEnable",
        "disable": "onDisable"
    }
}
```

### 6.3 loading_order 说明

- `loading_order` 值越小的插件越先加载
- 内置扩展使用较小的值（如 1~50）
- 第三方插件建议使用 50+ 避免与内置扩展冲突
- 同一顺序的插件按显示名称排序

## 七、常见问题

### Q1: 点击图标没有反应

**原因：** 插件没有为 `.drawer-toggle` 绑定点击事件。

**解决：** 必须显式绑定：
```javascript
$('#my-plugin-drawer .drawer-toggle').on('click', async function() {
    const { doNavbarIconClick } = await import('../../../../script.js');
    doNavbarIconClick.call(this);
});
```

### Q2: 入口图标不显示

**原因：** FontAwesome 图标类名错误或 CSS 未加载。

**检查：**
- 图标类名是否正确（如 `fa-solid fa-star`）
- `css` 文件是否在 `manifest.json` 中声明
- 图标是否有 `fa-fw` 类（固定宽度）

### Q3: 插入位置不对

**原因：** 插入方法或锚点选择器不正确。

**解决：** 使用两步法：先 `append` 确保添加成功，再 `insertBefore` 移动到目标位置。

### Q4: 刷新页面后入口丢失

**检查：**
- `manifest.json` 的 `hooks.activate` 是否正确指向 `init` 函数
- `init()` 函数中是否调用了 `addNavBarDrawer()`
- 浏览器控制台是否有 JavaScript 错误
