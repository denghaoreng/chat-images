# manifest.json 清单文件与生命周期钩子

## manifest.json 完整字段

`manifest.json` 是每个插件的元数据文件，ST 在加载插件时首先读取它。完整结构如下：

```json
{
    "display_name": "插件显示名称",
    "loading_order": 9,
    "js": "index.js",
    "css": "style.css",
    "author": "作者名",
    "version": "1.0.0",
    "homePage": "https://github.com/your/extension",
    "auto_update": true,
    "minimum_client_version": "1.0.0",
    "requires": ["summarize"],
    "optional": ["caption"],
    "dependencies": ["vectors", "third-party/Extension-WebLLM"],
    "generate_interceptor": "myInterceptorFunction",
    "i18n": {
        "de-de": "i18n/de-de.json",
        "zh-cn": "i18n/zh-cn.json"
    },
    "hooks": {
        "activate": "init",
        "install": "onInstall",
        "update": "onUpdate",
        "delete": "onDelete",
        "enable": "onEnable",
        "disable": "onDisable",
        "clean": "onClean"
    }
}
```

### 字段说明

| 字段 | 必需 | 说明 |
|------|------|------|
| `display_name` | ✅ | 扩展管理菜单中显示的插件名 |
| `js` | ✅ | 主 JS 入口文件路径 |
| `css` | ❌ | 样式文件路径 |
| `loading_order` | ❌ | 加载顺序，数字越大加载越晚（默认按名称排序） |
| `author` | ✅ | 作者名称或联系方式 |
| `version` | ❌ | 版本号 |
| `homePage` | ❌ | 项目主页 URL |
| `auto_update` | ❌ | 是否随 ST 包版本更新而自动更新 |
| `minimum_client_version` | ❌ | 最小 ST 客户端版本要求 |
| `requires` | ❌ | **已废弃**。需要的 Extras API 模块列表 |
| `optional` | ❌ | **已废弃**。可选的 Extras API 模块列表 |
| `dependencies` | ❌ | 依赖的其他插件（按目录名） |
| `generate_interceptor` | ❌ | 生成拦截器函数名（全局函数） |
| `i18n` | ❌ | 国际化配置，key=语言代码，value=JSON 文件路径 |
| `hooks` | ❌ | 生命周期钩子配置 |

### 🔍 深入解析：以 Memory 插件为例

真实的 `memory/manifest.json`：

```json
{
    "display_name": "Summarize",
    "loading_order": 9,
    "requires": [],
    "optional": ["summarize"],
    "js": "index.js",
    "css": "style.css",
    "author": "Cohee#1207",
    "version": "1.0.0",
    "homePage": "https://github.com/SillyTavern/SillyTavern",
    "hooks": {
        "activate": "init"
    }
}
```

关键点解读：
- `loading_order: 9` — 在中等偏后的位置加载（数字越大越晚）
- `optional: ["summarize"]` — 如果 Extras API 启用了 summarize 模块则可用，但不是必需的
- `hooks.activate: "init"` — 激活时调用 `index.js` 中导出的 `init` 函数

### dependencies 依赖规则

依赖按目录名指定（即 `public/scripts/extensions/` 下的文件夹名）：

- **内置插件**: `"vectors"`, `"caption"`
- **第三方插件**: `"third-party/Extension-WebLLM"`, `"third-party/Extension-Mermaid"`

如果依赖缺失或已被禁用，插件将无法加载。

## 生命周期钩子（Lifecycle Hooks）

钩子系统允许插件在特定的生命周期点执行代码。在 `manifest.json` 的 `hooks` 字段中配置后，在 JS 入口文件中导出对应的函数即可。

### 可用钩子一览

| 钩子名 | 触发时机 | 典型用途 |
|--------|----------|----------|
| `activate` | 插件成功激活时（页面加载时） | 注册事件监听、加载设置、添加 UI 控件 |
| `install` | 插件安装完成且设置加载后 | 首次安装时的初始化 |
| `update` | 插件成功更新后（刷新页面之前） | 数据迁移、格式升级 |
| `delete` | 插件从服务器删除前 | 清理存储数据、释放资源 |
| `enable` | 插件启用前 | 恢复功能 |
| `disable` | 插件禁用前 | 暂停功能、保存状态 |
| `clean` | 用户点击"清理扩展数据"按钮时 | 清理缓存数据 |

### 实现方式

```javascript
// index.js — 插件的入口文件

// activate 钩子：页面加载时自动调用
export async function init() {
    console.log('插件已激活');
    // 添加 UI 控件
    const html = await renderExtensionTemplateAsync('my-extension', 'settings', {});
    $('#extensions_settings').append(html);
    // 加载设置
    loadSettings();
    // 注册事件
    eventSource.on(event_types.CHAT_CHANGED, onChatChanged);
}

// install 钩子：安装时调用
export async function onInstall() {
    console.log('插件已安装，执行首次设置...');
    // 例如：初始化默认数据
}

// update 钩子：更新后调用
export async function onUpdate() {
    console.log('插件已更新，运行迁移...');
    // 例如：将旧格式数据迁移到新格式
}

// delete 钩子：删除前调用
export async function onDelete() {
    console.log('插件即将被删除，清理中...');
    // 例如：删除 localStorage 中的数据
    const { localforage } = SillyTavern.libs;
    await localforage.removeItem('my_extension_data');
}

// enable/disable 钩子：启用/禁用时调用
export function onEnable() { console.log('插件已启用'); }
export function onDisable() { console.log('插件已禁用'); }

// clean 钩子：清理数据
export async function onClean() {
    console.log('插件数据已清理');
}
```

### 钩子的执行机制

1. **所有钩子都是可选的**，不需要全部实现
2. 钩子函数可以返回 **Promise**（会被 await）
3. 钩子有 **5 秒超时**，超时后会继续执行并记录警告
4. 钩子中的错误会被捕获并记录，不会阻塞操作

### 🔍 Memory 插件的钩子实践

Memory 插件只使用了 `activate` 钩子：

```javascript
// manifest.json
"hooks": { "activate": "init" }

// index.js
export async function init() {
    // 1. 渲染设置面板
    await addExtensionControls();

    // 2. 加载已保存的设置
    loadSettings();

    // 3. 注册事件监听
    eventSource.on(event_types.CHAT_CHANGED, onChatChanged);
    eventSource.makeLast(event_types.CHARACTER_MESSAGE_RENDERED, onChatEvent);

    // 4. 注册斜杠命令
    SlashCommandParser.addCommandObject(SlashCommand.fromProps({
        name: 'summarize',
        callback: summarizeCallback,
        // ...
    }));

    // 5. 注册宏
    macros.register('summary', { ... });
}
```

## 插件加载机制（源码解析）

来自 `extensions.js` 的 `activateExtensions()` 函数说明了插件的完整加载流程：

```
1. 读取所有插件的 manifest.json
2. 按 loading_order 排序
3. 对每个插件依次检查：
   ├─ minimum_client_version ≥ 当前 ST 版本？
   ├─ requires 模块是否都可用？（已废弃）
   ├─ dependencies 依赖是否存在且未禁用？
   ├─ 插件本身未被禁用？
   └─ 全部通过 → 开始激活
4. 激活过程：
   ├─ 加载国际化数据（如果 manifest 配置了 i18n）
   ├─ 添加 CSS 样式文件（如果 manifest 配置了 css）
   ├─ 添加 JS 脚本文件（使用 <script type="module">）
   ├─ 标记为已激活
   └─ 调用 activate 钩子函数
```

如果插件因为依赖缺失或版本不满足而无法加载，ST 会在扩展管理菜单中显示警告信息。

## 实践技巧：导航栏 Drawer 的添加

在 `init` 钩子中添加导航栏抽屉时，推荐**两步法**确保正确插入位置：

```javascript
// 1. 先添加到末尾（保证入口一定存在）
$('#top-settings-holder').append(drawerHtml);

// 2. 再移动到目标位置（用户设定管理前面）
$('#chat-images-drawer').insertBefore('#user-settings-button');
```

**为什么两步法？** 如果直接使用 `insertBefore`，目标元素可能尚未渲染完毕，导致抽屉丢失。先 append 保证元素已存在再移动位置。

### 标签页状态记忆

如果抽屉中有多个标签页，可以用 `window.lastActiveTab` 记录用户最后使用的标签页：

```javascript
// 标签切换时记录
window.lastActiveTab = tab;

// 打开抽屉时恢复
const tab = window.lastActiveTab || 'charsets'; // 默认第一个标签

// 刷新页面后重置
window.lastActiveTab = null;
```
