# SillyTavern 插件开发总览

## 什么是 SillyTavern 插件？

SillyTavern（简称 ST）是一个面向 LLM 用户的本地界面。插件（Extensions）是扩展 SillyTavern 功能的主要方式，它们运行在浏览器环境中，拥有对 DOM、JavaScript API 和 SillyTavern 上下文（Context）的几乎不受限制的访问权限。

> 📖 **官方文档**: https://docs.sillytavern.app/for-contributors/writing-extensions/

## 插件架构

SillyTavern 的插件系统分为两大部分：

### 1. 前端 UI 扩展（UI Extensions）
- 运行在浏览器中，使用 JavaScript 编写
- 可以操作 DOM、监听事件、修改 UI
- 通过 `getContext()` API 与 ST 核心交互
- 文件位于 `public/scripts/extensions/` 目录下

### 2. 服务端插件（Server Plugins）
- 运行在 Node.js 服务端
- 扩展 ST 后端的 API 和功能
- 使用不同的机制

> 本文档聚焦于**前端 UI 扩展**开发。

## 插件目录结构

一个标准的 SillyTavern 插件包含以下文件：

```
my-extension/
├── manifest.json      # 插件清单（必需）
├── index.js           # 主 JS 入口（必需）
├── style.css          # 样式文件（可选）
├── settings.html      # 设置面板 HTML 模板（可选）
└── i18n/              # 国际化文件（可选）
    └── zh-cn.json
```

### 模块拆分建议

插件只需一个 `index.js` 作为入口，但**复杂插件建议拆分为多模块**以保持代码可维护性：

```
my-extension/
├── manifest.json      # 引用 index.js
├── index.js           # 入口：生命周期、事件、导入
├── data.js            # 数据层：CRUD 操作
├── ui.js              # UI 渲染
├── utils.js           # 工具函数
├── style.css
└── settings.html
```

拆分方式：入口文件 `index.js` 使用 ES Module `import` 导入子模块，`manifest.json` 只引用 `index.js`：

```javascript
// index.js
import { loadSettings, saveSettings } from './data.js';
import { renderUI } from './ui.js';
```

SillyTavern 会以 `<script type="module">` 方式加载入口文件，因此模块内 `import` 语句可以正常使用。

## 两种安装方式

### 1. 为所有用户安装（服务端级）
将插件仓库放在 `public/scripts/extensions/third-party/` 目录下。要求重启 SillyTavern 服务端。

### 2. 为当前用户安装（用户级）
通过 SillyTavern 的 "Download Extensions & Assets" 菜单下载安装。数据存放在 `data/<user-handle>/extensions/` 目录下。

> 本地开发时，建议将插件放在 `public/scripts/extensions/third-party/` 目录下以简化路径引用。

---

## 快速开始

1. 在 `public/scripts/extensions/third-party/` 下创建你的插件文件夹
2. 创建 `manifest.json` 和 `index.js`
3. 在 `index.js` 中导出 `init` 函数（由 `activate` 钩子调用）
4. 使用 `renderExtensionTemplateAsync()` 加载 HTML 设置面板
5. 使用 `eventSource.on()` 监听事件
6. 重启 SillyTavern 或刷新页面查看效果
