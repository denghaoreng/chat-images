# 事件系统与状态管理

## 事件系统（Events）

SillyTavern 使用 `EventEmitter` 实现了一套强大的事件系统，插件可以通过监听事件来响应应用的各种状态变化。

### 获取事件源

```javascript
const { eventSource, event_types } = SillyTavern.getContext();
// 或通过 import
import { eventSource, event_types } from '../../../script.js';
```

### 监听事件

```javascript
eventSource.on(event_types.MESSAGE_RECEIVED, handleIncomingMessage);

function handleIncomingMessage(data) {
    console.log('收到新消息', data);
}
```

### 一次性监听

```javascript
eventSource.once(event_types.APP_READY, () => {
    console.log('应用准备就绪 - 只触发一次');
});
```

### 移除监听

```javascript
eventSource.removeListener(event_types.MESSAGE_RECEIVED, handleIncomingMessage);
```

### 发射事件（供插件自定义）

```javascript
const { eventSource } = SillyTavern.getContext();
await eventSource.emit('myCustomEvent', { data: '自定义数据' });
```

使用 `await` 确保所有事件处理器完成后再继续执行。

### 使用 makeLast 确保最后执行

```javascript
// 让此监听器最后执行（在同类事件的其他监听器之后）
eventSource.makeLast(event_types.CHARACTER_MESSAGE_RENDERED, onChatEvent);
```

> 🔍 Memory 插件使用 `makeLast` 来确保在消息渲染完成后再触发总结检查。

## ⚠️ 实战陷阱：弹窗（Popup）DOM 生命周期

使用 `callGenericPopup()` 创建弹窗时，**弹窗关闭后其 DOM 元素会被销毁**，之前获取的输入框引用会失效。

### 错误做法

```javascript
const popupHtml = `<input id="my-input" type="text">`;
const popup = await callGenericPopup(popupHtml, POPUP_TYPE.CUSTOM, { ... });

// ❌ 弹窗关闭后 DOM 已销毁，以下取值返回 undefined
const value = $('#my-input').val();
```

### 正确做法：实时监听

```javascript
// 在弹窗显示之前定义变量
let myValue = '';

const popupHtml = `
    <input id="my-input" type="text" placeholder="输入内容">
`;

// 先注入 HTML，等待 DOM 生成，再绑定事件
$('body').on('input', '#my-input', function() {
    myValue = $(this).val();
});

const popup = await callGenericPopup(popupHtml, POPUP_TYPE.CUSTOM, { ... });

// ✅ 此时 myValue 已被实时更新，可正常使用
if (popup) {
    console.log('用户输入:', myValue);
}
```

### 替代方案：OK/CANCEL 回调

也可以使用 `callGenericPopup` 的 `okButton` 和 `cancelButton` 回调参数：

```javascript
const popupHtml = `<input id="my-input" class="text_pole" type="text">`;

const popup = await callGenericPopup(popupHtml, POPUP_TYPE.CUSTOM, {
    okButton: '确定',
    cancelButton: '取消',
    // 确定按钮回调中取值
    onOk: function() {
        return $('#my-input').val() || '';
    }
});
```

> 注意：`onOk` 回调在弹窗关闭前执行，此时 DOM 尚未销毁，可以正常读取值。

### 完整事件列表

#### 应用生命周期

| 事件常量 | 说明 |
|----------|------|
| `APP_INITIALIZED` | 应用初始化完成，但加载器仍可见。适合做 UI 修改。**自动触发**给新监听器。 |
| `APP_READY` | 应用完全加载，可正常使用。**自动触发**给新监听器。 |
| `EXTRAS_CONNECTED` | Extras API 连接成功，参数为模块列表 |

#### 消息事件

| 事件常量 | 说明 |
|----------|------|
| `MESSAGE_SENT` | 用户发送消息，已记录到 chat 但未渲染到 UI |
| `MESSAGE_RECEIVED` | LLM 消息已生成，已记录但未渲染 |
| `USER_MESSAGE_RENDERED` | 用户消息已渲染到 UI |
| `CHARACTER_MESSAGE_RENDERED` | LLM 消息已渲染到 UI |
| `MESSAGE_EDITED` | 消息被编辑 |
| `MESSAGE_DELETED` | 消息被删除 |
| `MESSAGE_SWIPED` | 消息滑动（切换回复） |
| `MESSAGE_UPDATED` | 消息更新 |
| `STREAM_TOKEN_RECEIVED` | 流式生成中收到新 token |
| `MORE_MESSAGES_LOADED` | 更多历史消息被加载 |

#### 生成事件

| 事件常量 | 说明 |
|----------|------|
| `GENERATION_AFTER_COMMANDS` | 即将开始生成（斜杠命令处理后） |
| `GENERATION_STARTED` | 生成已开始 |
| `GENERATION_STOPPED` | 用户停止了生成 |
| `GENERATION_ENDED` | 生成已完成或报错 |

### 初始化最佳实践

```javascript
export async function init() {
    // 1. activate 钩子：同步设置，加载器仍在显示
    // 适合：注册监听器、加载设置

    // 2. APP_INITIALIZED：所有插件和 UI 已加载，但加载器仍可见
    eventSource.on(event_types.APP_INITIALIZED, () => {
        // 适合：UI 修改
    });

    // 3. APP_READY：应用完全可用
    eventSource.on(event_types.APP_READY, () => {
        // 适合：异步初始化，不阻塞应用
    });
}
```

## 状态管理

### 1. 持久化设置（Persistent Settings）

插件通过 `extensionSettings` 对象存储和读取数据：

```javascript
const { extensionSettings, saveSettingsDebounced } = SillyTavern.getContext();

// 定义唯一模块名
const MODULE_NAME = 'my_extension';

// 定义默认设置（使用 Object.freeze 防止意外修改）
const defaultSettings = Object.freeze({
    enabled: false,
    option1: 'default',
    option2: 5,
});

// 加载设置
function loadSettings() {
    if (!extensionSettings[MODULE_NAME]) {
        extensionSettings[MODULE_NAME] = structuredClone(defaultSettings);
    }
    // 确保所有默认键存在（在更新后很有用）
    for (const key of Object.keys(defaultSettings)) {
        if (extensionSettings[MODULE_NAME][key] === undefined) {
            extensionSettings[MODULE_NAME][key] = defaultSettings[key];
        }
    }
}

// 使用设置
const settings = extensionSettings[MODULE_NAME];
settings.option1 = 'new value';

// 保存设置（防抖）
saveSettingsDebounced();
```

#### 🔍 Memory 插件的设置管理

```javascript
const MODULE_NAME = '1_memory';  // 注意：使用前缀确保唯一性

const defaultSettings = {
    memoryFrozen: false,
    SkipWIAN: false,
    source: 'extras',
    prompt: 'Ignore previous instructions...',
    template: '[Summary: {{summary}}]',
    position: 0,          // extension_prompt_types.IN_PROMPT
    role: 0,              // extension_prompt_roles.SYSTEM
    scan: false,
    depth: 2,
    promptWords: 200,
    promptInterval: 10,
    promptForceWords: 0,
    // ...更多设置
};

function loadSettings() {
    if (Object.keys(extension_settings.memory).length === 0) {
        Object.assign(extension_settings.memory, defaultSettings);
    }
    for (const key of Object.keys(defaultSettings)) {
        if (extension_settings.memory[key] === undefined) {
            extension_settings.memory[key] = defaultSettings[key];
        }
    }
    // 然后同步到 UI 控件
    $('#summary_source').val(extension_settings.memory.source).trigger('change');
    // ...
}
```

> **安全警告**: 绝不要在 `extensionSettings` 中存储 API 密钥或密码！所有扩展都可以访问这些设置。

### 2. 聊天元数据（Chat Metadata）

将数据绑定到特定聊天：

```javascript
const { chatMetadata, saveMetadata } = SillyTavern.getContext();

// 设置当前聊天的元数据
chatMetadata['my_key'] = 'my_value';

// 读取元数据
const value = chatMetadata['my_key'];

// 保存到服务器
await saveMetadata();
```

> 不要长期持有 `chatMetadata` 引用——切换聊天后引用会变化。始终使用 `SillyTavern.getContext().chatMetadata` 获取当前值。监听 `CHAT_CHANGED` 事件来同步状态。

### 3. 角色卡片数据（Character Card Data）

利用 Character Card V2 规范的 `extensions` 字段存储数据：

```javascript
const { writeExtensionField, characterId } = SillyTavern.getContext();

// 写入角色卡片
await writeExtensionField(characterId, 'my_extension_key', {
    someData: 'value',
    anotherData: 42,
});

// 读取角色卡片数据
const character = SillyTavern.getContext().characters[characterId];
const myData = character.data?.extensions?.my_extension_key;
```

> ⚠️ `characterId` 是一个索引而非真正唯一标识符。在群聊或未选择角色时为 `undefined`。

### 4. 设置预设数据（Settings Presets）

将数据存储到 API 预设中：

```javascript
const { getPresetManager } = SillyTavern.getContext();
const pm = getPresetManager();

// 写入预设扩展字段
await pm.writePresetExtensionField({ path: 'hello', value: 'world' });

// 读取预设扩展字段
const value = pm.readPresetExtensionField({ path: 'hello' });
```

监听 `PRESET_CHANGED` 和 `MAIN_API_CHANGED` 事件来同步状态。

### 5. 使用 localforage 存储大数据

对于大数据，不要存在 `extensionSettings` 中：

```javascript
const { localforage } = SillyTavern.libs;

// 存储
await localforage.setItem(`${MODULE_NAME}_data`, largeData);

// 读取
const data = await localforage.getItem(`${MODULE_NAME}_data`);

// 删除
await localforage.removeItem(`${MODULE_NAME}_data`);
```

## 🔍 Memory 插件的事件使用模式

Memory 插件展示了多个事件组合使用的经典模式：

```javascript
export async function init() {
    // ... 加载 UI 和设置 ...

    // 聊天切换时：更新显示最新的总结
    eventSource.on(event_types.CHAT_CHANGED, onChatChanged);

    // 消息渲染后：检查是否需要执行总结
    eventSource.makeLast(event_types.CHARACTER_MESSAGE_RENDERED, onChatEvent);

    // 消息删除、编辑、滑动后：也检查总结
    for (const event of [event_types.MESSAGE_DELETED, event_types.MESSAGE_UPDATED, event_types.MESSAGE_SWIPED]) {
        eventSource.on(event, onChatEvent);
    }
}
```

`onChatEvent` 中的条件判断：

```javascript
async function onChatEvent() {
    // 1. 模块未启用
    if (source === 'extras' && !modules.includes('summarize')) return;
    // 2. WebLLM 不支持
    if (source === 'webllm' && !isWebLlmSupported()) return;
    // 3. 流式生成进行中
    if (streamingProcessor && !streamingProcessor.isFinished) return;
    // 4. 正在总结中或冻结状态
    if (inApiCall || extension_settings.memory.memoryFrozen) return;
    // 5. 无新消息
    if (lastMessageId === chat.length && hash === lastMessageHash) return;

    summarizeChat(context);
}
```
